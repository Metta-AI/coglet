# PCO LLM Experiment — Code Generation

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Run a real LLM-powered PCO loop where the actor generates Python solutions to 15 programming puzzles, the critic predicts pass/fail, and the learner improves both over 5 epochs. Measure actor pass rate and critic accuracy improvement.

**Architecture:** All components are coglets using the PCO framework. Actor holds solutions as `dict[str, str]`, critic holds an evolving evaluation strategy, learner makes Claude Sonnet calls to rewrite failing solutions and update critic strategy.

**Tech Stack:** Python 3.11+, coglet PCO framework, Anthropic API (Claude Sonnet), pytest

---

## Components

### Actor (CodeGenActor)
- State: `dict[str, str]` — puzzle name → Python source code
- `enact("run")`: execute each solution against test cases, transmit experience
- `enact("update")`: replace specific solutions from learner's patch
- Initial solutions generated by one LLM call at epoch 0

### Critic (CodeReviewCritic)
- State: `critic_strategy: str` — evolving evaluation approach
- `listen("experience")`: read solutions (not test results), predict pass/fail per puzzle
- `enact("update")`: update strategy from learner's patch
- Starts with generic strategy: "Look for common bugs"

### Learner (CodeGenLearner)
- Receives: experience (test results), evaluation (predictions vs actuals), signals
- LLM call: "Fix failing solutions (max 5). Update critic strategy from its mistakes."
- Transmits: `{"solutions": {...}, "critic_strategy": "..."}`

### Losses
- `ActorLoss` — number of failing puzzles
- `CriticLoss` — number of wrong predictions

### Constraints
- `ChangeMagnitude` — reject if >5 solutions rewritten at once

## Puzzle Suite (15 problems)

**Easy (5):** fizzbuzz, is_palindrome, reverse_words, max_of_list, factorial
**Medium (5):** balanced_parens, roman_to_int, run_length_encode, flatten_list, nth_prime
**Hard (5):** eval_rpn, longest_common_subseq, spiral_matrix, merge_intervals, valid_sudoku

## Metrics

Per epoch:
- `actor_pass_rate` — fraction of puzzles passing all tests
- `critic_accuracy` — fraction of correct pass/fail predictions
- `critic_fp` — predicted pass, actually failed
- `critic_fn` — predicted fail, actually passed

Assertions:
1. `actor_pass_rate[last] > actor_pass_rate[first]`
2. `critic_accuracy[last] > critic_accuracy[first]`
3. `actor_pass_rate[last] >= 10/15`

Results written to `.tests/pco_experiment/` (gitignored).

## LLM Calls

3 per epoch × 5 epochs = 15 calls (Claude Sonnet). ~2 min total.
